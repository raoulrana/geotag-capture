# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A dependency-free vanilla-JS PWA (`www/`) that turns a phone/laptop camera into a geotagging
camera: every capture gets a panel stamped onto it with lat/long, reverse-geocoded address,
time, an optional mini OSM map, and an optional remark — with real EXIF GPS metadata embedded
in the saved JPEG. It's wrapped in Capacitor for iOS/Android native builds; the web version
doubles as a demo/marketing funnel.

## Commands

```bash
npm run serve        # serve www/ at http://localhost (via `serve`); camera/geo need a secure context
npm run icons         # regenerate PNG app icons from generate-icons.js
node test-exif.js     # round-trips the EXIF writer through exifr and asserts GPS/date/etc match
npm run sync           # cap sync — push www/ changes into the native ios/ project
npm run open:ios       # cap open ios — open the Xcode project
npm run open:android   # cap open android
bash scripts/setup-ios.sh   # one-shot: installs CocoaPods, `cap add ios`, injects Info.plist
                              # permission strings, cap sync, opens Xcode
```

There is no bundler, transpiler, or lint/test framework — `www/*.js` is loaded directly as
plain `<script>` tags in `www/index.html`, in dependency order (`exif.js`, `olc.js`, then
`app.js`). Edit and reload; no build step for the web app itself.

To test on a real phone before the native build exists (camera/GPS require HTTPS or
`localhost`): serve `www/` locally and tunnel it (e.g. `cloudflared tunnel --url
http://localhost:8000`), then open the HTTPS URL on-device — Wi-Fi/IP geolocation on a laptop
is too coarse to validate GPS behavior.

`ios/` and `android/` are git-ignored — they're regenerated by `cap add`/`cap sync`, not
hand-edited (aside from the permission strings `setup-ios.sh` injects into `Info.plist`).

## Architecture

Everything client-side lives in `www/`:

- **`app.js`** (~1400 lines) — the entire app: camera stream, geolocation watch, reverse
  geocoding/postcode lookup, mini-map tile rendering, capture compositing, IndexedDB capture
  log, history/map/settings views, CSV/GeoJSON export, native bridging. There's no module
  system — it's one file with top-level functions operating on two shared objects:
  - `els` — cached DOM element references, one per view.
  - `state` — mutable runtime state (current GPS fix, map tile, loaded settings, etc).
  - `settings` — persisted to `localStorage`, loaded via `loadSettings()`/`saveSettings()`
    against `DEFAULT_SETTINGS`; drives stamp template, accent color, per-field toggles, logo.
- **`exif.js`** — standalone, dependency-free EXIF/TIFF writer (`window.GeoExif.embedExif`).
  Builds the IFD0/Exif/GPS blocks and splices an APP1 segment into a JPEG right after SOI.
  Don't hand-roll EXIF byte-fiddling elsewhere — extend this module. Verified against `exifr`
  via `test-exif.js`.
- **`olc.js`** — vendored, unmodified Google Open Location Code (Plus Codes) reference impl,
  wrapped in a browser shim. Treat as third-party: don't hand-edit.
- **`service-worker.js`** — network-first app-shell cache (deliberately not cache-first — an
  earlier cache-first version caused stale-code confusion during device testing). Map tiles
  and geocoding lookups are network-only.
- **`manifest.webmanifest`** — PWA install metadata.

### Native bridge (`isNative` in app.js)

`isNative = !!(window.Capacitor && window.Capacitor.isNativePlatform())`. When true, GPS comes
from `Capacitor.Plugins.Geolocation` (aliased `CapGeo`) and saves go through
`Capacitor.Plugins.Filesystem` (`CapFS`) instead of the browser `getUserMedia`/download path —
check both branches when touching capture, geolocation, or save/share code. Camera permissions
and geolocation permissions are requested up front in `init()` on native, not lazily at capture
time. `capacitor.config.json` sets `appId: com.geotagcapture.app`, `webDir: www`.

### Capture pipeline

Live `<video>` frame → drawn to `captureCanvas` at full sensor resolution → geotag panel
(coords, address, time, remark, re-rendered mini map) composited on top via
`composeCapture()`/`drawGeotagPanel()` (three template variants: card/bar/minimal, see
`drawBottomPanel`/`drawMinimal`) → exported as JPEG blob → EXIF GPS embedded via
`GeoExif.embedExif` → logged to IndexedDB (`logCapture`) → save/share via native or browser
path. The on-screen overlay (`renderOverlay`) is a live preview only — the stamp is re-rendered
from scratch onto the canvas, not screenshotted.

### Data log (IndexedDB)

`openDB()`/`dbTx()` wrap a single IndexedDB store of past captures (blob + full metadata:
coords, address, altitude/speed/heading, weather, Plus Code, project, tags, remark). Feeds the
history gallery, the captures map view (`renderCapturesMap()` — a hand-rolled canvas multi-tile
OSM renderer, no Leaflet/Mapbox dependency), and CSV/GeoJSON export.
